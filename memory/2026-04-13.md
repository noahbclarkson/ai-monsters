## Findings (3 things to improve)

### 1. Pack card reveal animation feels slow and staggered
- **Which component:** PackOpening.tsx
- **What's wrong:** Cards appeared all at once (revealedCount set to 5), then the setTimeout loop was incrementing at 400ms+220ms intervals — the 400ms base delay meant cards didn't start flipping until 400ms after showing
- **Why it matters:** The "staggered flip" is the core emotional moment of pack opening — it should feel snappy and satisfying
- **How to fix:** Reduced initial delay (300ms) and stagger (180ms) — already committed as d5de9e2

### 2. Image gen API not working in production
- **Which component:** `/api/generate-card-image` route, also `/api/generate-description`
- **What's wrong:** MINIMAX_API_KEY and OPENAI_API_KEY not set in production .env — API returns gradient fallbacks instead of AI-generated art/descriptions. The gradient art system is actually beautiful (SVG data URIs with rarity colors, type shapes, initials), but AI content is what makes the game distinctive
- **Why it matters:** Player sees generic gradient art instead of unique AI monsters — undermines the core "AI-generated" hook
- **How to fix:** Need to configure real API keys in production environment. The gradient fallback is a good safety net but the real API calls need to succeed

### 3. Daily card generator has no loading state on button
- **Which component:** DailyCardGenerator.tsx
- **What's wrong:** The "Claim Today's Card" button uses `disabled={generating}` but `generating` is a local state that controls the whole flow. When clicked, the generating state is set but there's no inline loading feedback on the button itself — only a spinner in the card slot
- **Why it matters:** User clicks button and doesn't get immediate tactile feedback — the whole UI just sits there while cards are generated (which takes 1-2 seconds)
- **How to fix:** Add loading state indicator to the button itself (spinner + text change to "Generating...")
