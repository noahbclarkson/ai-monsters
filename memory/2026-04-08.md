### Issues Found

**Issue 1: Leaderboard.tsx - White/Light Theme (MOST CRITICAL)**
- Component: `/client/src/components/Leaderboard.tsx`
- Problem: Entire component uses white/gray backgrounds and light text
  - Main container: `bg-white rounded-lg shadow-lg p-6`
  - Sort buttons: `bg-blue-500 text-white`, `bg-gray-100 text-gray-700`
  - Player rows: `bg-gray-50`, `bg-gradient-to-r from-yellow-50 to-amber-50`
  - Rank badges: `bg-purple-100`, `bg-red-100` (light mode colors)
  - Text: `text-gray-800`, `text-gray-900` (light mode text)
- Why it matters: Leaderboard is visible from the main nav. White background
  breaks immersion completely on a dark-themed game. First impression is
  "this looks like a prototype" even though the rest of the UI is polished.
- Fix: Complete rewrite with glass-morphism, dark palette, rank-specific glows

**Issue 2: CollectionGallery.tsx - Header/stats area not glass-morphism**
- Component: `/client/src/components/CollectionGallery.tsx`
- Problem: The header section uses `bg-gradient-to-br from-gray-900 via-purple-900 to-blue-900`
  and plain `bg-black/30` rounded cards for stats. The rest of the app uses
  glass-card with backdrop-blur and premium borders. This section stands out
  as "still needs polish."
- Why it matters: The collection is a core feature players will visit often.
  Stats and header should feel as premium as the rest of the UI.
- Fix: Wrap header in glass-card, use glass-morphism for stats bar

**Issue 3: MainNavigation.tsx - Nav button active state very subtle**
- Component: `/client/src/components/MainNavigation.tsx`
- Problem: Active tab gets only a 2px white bottom border. In certain lighting
  or on lower-quality displays, this could be hard to notice. The buttons
  themselves use `text-white bg-white/10` for active state which is subtle.
- Why it matters: Users need to always know which tab is active.
- Fix: Add stronger active state - active glow, brighter text, more contrast.

**Issue 4: Navigation emoji icons in nav buttons (low priority)**
- MainNavigation nav buttons: use `text-xs text-white` with lucide icons correctly
  (the icon is a React component). But the aria-label text is set via
  `{tab.label}` which would render as text, not emoji. Looking more closely,
  the nav tabs use lucide SVG icons correctly - no emoji in the nav buttons
  themselves. So this was a false alarm from the accessibility tree.
