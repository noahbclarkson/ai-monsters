## 2026-04-09: Implementation cycle (18:22 UTC)

### Audit findings
- Card generation: AICardGenerator.generateCardImage was returning fake `/api/generated/...` paths instead of calling MiniMax API
- Description generation: generate-description route returned 500 when OpenAI API key missing/invalid, instead of using fallback
- Mobile layout: difficulty buttons in GameLobby were 3-column on mobile (too cramped)
- Mobile nav: tab labels were hidden on mobile (only icons showed)

### Fixes applied
1. **ai-card-generator.ts**: Replaced fake `generateCardImage` with actual `/api/generate-card-image` API call, falls back to picsum.photos on failure
2. **generate-description/route.ts**: Changed to return fallback description when OpenAI API key missing or API fails (was returning 500)
3. **generate-description/route.ts**: Made fallback descriptions varied (3 variants per type+rarity combo, randomly picked)
4. **GameLobby.tsx**: Changed difficulty grid from `grid-cols-3` to `grid-cols-1 sm:grid-cols-3` for mobile stacking
5. **MainNavigation.tsx**: Changed nav tab labels from `hidden sm:inline` to `text-xs sm:text-sm` so they always show

### Build status
- npm run build: PASS
- cargo check: PASS

- Pack opening is slow (~30s per card, ~2.5min for 5 cards) due to sequential MiniMax API calls
- OpenAI API key is invalid (401) — descriptions use fallback, which is functional but less creative
